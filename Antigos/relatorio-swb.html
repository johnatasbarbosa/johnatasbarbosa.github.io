Resumo: Este relatório descreve as ferramentas necessárias e adicionais utilizadas durante o desenvolvimento do código fonte de um programa utilizando a linguagem assembly. Nele é descrito cada uma destas ferramentas, indicando como instalar e informações básicas sobre a sua utilidade.
Palavras-chave: Desenvolvimento, Assembly, Ferramentas.

1. Introdução

Para que programas sejam executados em um computador, o mesmo deve ser representado em linguagem de máquina, 0s e 1s, única forma em que é compreendido. Porém, esta linguagem não é compreensível ao ser humano, sendo necessário uma linguagem mais compreensível ao ser humano e que possa ser transformada em linguagem de máquina, que representará dados e instruções de forma a realizar ações programadas no computador.

Para trabalhar com esta linguagem de programação são imprescindíveis algumas ferramentas e outras são importantes para melhor compreender o funcionamento da linguagem de máquina no computador. O presente relatório apresenta a instalação e funcionamento de cada programa.

2. Metodologia

Os programas foram instalados em um computador portando o sistema operacional Linux Mint 18.2 Cinnamon. A metodologia aplicada foi abordar cada programa individualmente, onde primeiro é descrito os comandos utilizados para a sua instalação e em seguida uma breve descrição e sua utilização.

$ sudo apt-get update

3. Processo de instalação e utilização

O processo de instalação foi feito via terminal, que pode ser aberto utilizando o atalho ctrl+alt+t'.

Primeiramente foi executado o comando abaixo para realizar a atualização do repositório.

$ sudo apt-get update

3.1 Editor de texto

Primeiramente deve-se definir o editor de texto que será utilizado. O mint possui o programa "Editor de texto" que existe de forma nativa, onde basta digitar o código fonte e salvar um arquivo com a extensão desejáda. Existem outros editores mais robostos, como o Atom e Visual Code, que possuem funcionalidades extras, como organização e listagem de diretório, função de auto completar e terminal integrado.

3.2 GCC

É uma suíte de compiladores gratuita que suporta muitas linguagens, como c/c++ e java.

Instalação

Já vem nativamente no linux mint. Caso não seja encontrado, pode ser instalado utilizando o comando abaixo:

$ sudo apt-get install gcc

Utilização

Com um código fonte feito em um editor, como por exemplo swb.c, a compilação e geração do executável é realizada com o seguinte comando:

$ gcc swb.c -o swb

O programa pode ser executado com o comando abaixo:

$ ./swb

3.3 GDB

É o debbuger do projeto GNU que permite visualizar o que está acontecendo dentro de um programa enquanto executa.

Instalação

Já vem nativamente no linux mint. Caso não seja encontrado, pode ser instalado utilizando o comando abaixo:

$ sudo apt-get install gdb

Utilização

Para sua utilização, durante o processo de compilação visto anteriormente, deve-se adicionar a flag –g, ficando o comando na forma:

$ gcc swb.c -o -g swb

E deve ser inicializado com o comando:

$ gdb swb

Alguns comandos básicos são:

  • run: este comando é utilizado para informar ao gdb que o programa deve ser inicializado.
  • kill: este comando força o termino da execução do programa.
  • quit: este comando realiza a saída em definitivo do gdb.

3.4 KDBG

É uma interface gráfica para o debbuger gdb que busca facilitar a visualização e acompanhamento das operações. Ele prover uma interface intuitiva para configurar breakpoints, inspecionar variáveis e avançar através do código.

Instalação

Instalando o kdbg com o comando abaixo:

$ sudo apt-get install kdbg

Utilização

É possível abrir o programa utilizando o atalho criado após a instalação ou simplesmente executando "kdbg" no terminal.

3.5 Nasm

É um software Assembly para arquitetura x86 que permite montar o código Assembly. Está disponível de forma livre para vários sistemas operacionais como Linux e Windows.

Instalação

Instalando o nasm com o comando abaixo:

$ sudo apt-get install nasm

Utilização

Com um arquivo nasm, q possui extensão .asm, criado, como exemplo um arquivo com o nome "swb.asm", basta executa o comando:

$ nasm -f elf swb.asm

Isso gerará o arquivo "swb.o" que é o arquivo de objeto. Para criar o executável para executar o seguinte comando:

$ ld swb.o -o swb

3.6 Objdump

É um programa que exibe várias informações sobre arquivos de objeto em sistemas semelhantes a Unix. Por exemplo, ele pode ser usado como um "disassembler", de forma a prover a visualização um executável em formato assembly.

Instalação

Existem de forma nativa em sistemas operacionais linux, mas caso não esteja instalado, basta executar o seguinte comando:

$ sudo apt-get install binutils

Utilização

Para a sua utilização é necessário a existência do executável do programa, que pode ser gerado como visto na subseção anterior e será utilizado como exemplo aqui.

A estrutura do comando é a seguinte:

$ Objdump –flag swb

Onde -flag indica o comando do objdump. Alguns deles são:

  • Exibir o conteúdo do cabeçalho geral do arquivo usando a opção -f
  • Exibir o conteúdo dos cabeçalhos de seção usando a opção -h
  • Exibir o conteúdo de todos os cabeçalhos usando a opção -x
  • Exibir o conteúdo do montador de todas as seções usando a opção -D
  • Exibir informações de debug usando a opção -g
  • Exibir o conteúdo completo de todas as seções usando a opção -s

3.6 BLESS EDITOR (EDITOR HEXADECIMAL)

É um programa que permite editar arquivos como uma sequência de bytes, sendo importante quando se está trabalhando com linguagem assembly, pois a mesma manipula endereços em hexadecimal.

Instalação

Instalando o bless editor com o comando abaixo:

$ sudo apt-get install bless

Utilização

É possível abrir o programa utilizando o atalho criado após a instalação ou simplesmente executando "bless" no terminal.

4. Conclusão

Conforme visto neste relatório, as ferramentas têm papel importante no desenvolvimento de software, fornecendo ao desenvolvedor meios de realizar operações no computador através de linguagem de montagem como em linguagens de níveis mais altos.

Sabe-se que no geral são utilizadas linguagens de alto no desenvolvimento de programas em geral, mas isso não tira a importância de se conhecer a linguagem assembly, principalmente em aplicações especificas e com necessidade de alto nível
